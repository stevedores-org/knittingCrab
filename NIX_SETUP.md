# Nix Setup & Attic Cache Configuration

## Quick Start

Enter the nix development environment:

```bash
nix develop
```

This provides a preconfigured Rust 1.93.1 toolchain with all dependencies for building and testing knittingCrab.

## Attic Cache Configuration

To use the `nix-cache.stevedores.org` attic cache for faster builds, you need to:

### Option 1: System-Wide Configuration (Recommended)

1. Get the public key from the cache administrator
2. Edit `/etc/nix/nix.conf` (requires `sudo`):

```nix
substituters = https://nix-cache.stevedores.org/ https://cache.nixos.org/
trusted-public-keys = nix-cache.stevedores.org:<PUBLIC_KEY> cache.nixos.org-1:6NCHdD59X431o0gWypG7a9Tv8sfqncan61pXZx54MKA=
```

3. Restart the nix daemon:
```bash
sudo launchctl stop org.nixos.nix-daemon
sudo launchctl start org.nixos.nix-daemon
```

### Option 2: Per-Project Setup (No Root Required)

For development without system-wide changes, the flake.nix is configured to work with the default nixos.org cache, which is sufficient for most use cases.

## Flake Structure

- `flake.nix`: Defines the dev environment with Rust toolchain
- `flake.lock`: Pins exact versions of dependencies (generated by nix)
- `.direnvrc`: Auto-loads nix environment when entering directory (requires direnv)

## Using Direnv (Optional)

For automatic environment loading when you `cd` into the directory:

```bash
# Install direnv if not already installed
brew install direnv

# Add to ~/.zshrc or ~/.bashrc:
eval "$(direnv hook zsh)"

# Allow the project directory
cd /path/to/knittingCrab
direnv allow
```

Then, whenever you enter the directory, the nix environment will automatically load.

## Building Without Nix

The project also builds with system Rust:

```bash
cargo test --workspace
cargo clippy --workspace -- -D warnings
```

## Cache Performance

- **With attic cache**: Pre-built binaries downloaded (~1-2 min for full build)
- **Without attic cache**: Builds from source (~5-10 min on M4 Max)
- **With system Rust**: Uses already-installed toolchain (~2-3 min)

## Troubleshooting

### "ignoring untrusted substituter" warning

This means the cache isn't configured in `/etc/nix/nix.conf`. Use Option 1 above to configure system-wide.

### Flake evaluation errors

Clear the flake cache:
```bash
rm flake.lock
nix flake update
```

### "nix: command not found"

Install nix:
```bash
curl --proto '=https' --tlsv1.2 -sSf -L https://install.determinate.systems/nix | sh -s -- install
```
